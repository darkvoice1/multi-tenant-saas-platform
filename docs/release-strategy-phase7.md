# 发布说明与版本策略（Phase 7）

## 版本号规范
- 采用 SemVer：`MAJOR.MINOR.PATCH`
- 示例：`v1.2.3`

## 分支与标签
- `main`：稳定主线
- 发布使用 Git tag：`vX.Y.Z`
- 每次发布都附带 Release Notes

## 发布说明模板
- 版本号：
- 发布时间：
- 变更摘要：
- 新增功能：
- 修复问题：
- 影响范围：
- 回滚方案：

## 建议流程
1. 功能合并到 `main`
2. 打 tag（如 `v1.1.0`）
3. 生成 Release Notes
4. 部署（本地构建或镜像拉取）
5. 记录回滚点（保留上一个稳定 tag）
